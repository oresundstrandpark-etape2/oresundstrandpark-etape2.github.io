HowToPost - Guide til (gratis) foreningshjemmesider på github

Nedenstående virker på Windows, Mac og Linux.

Alle i bestyrelsen kan redigere, der er historik, ændringer kan rules tilbage, det er globalt anvendt, tager 1-5 minutter at opdatere...

Github - Kun første gang

  1. Opret login til github. Click på link eller åben din browser og gå til github.com. Her kan du signe up med username, email, password, så du har en konto. Når du så har en konto, kan du bede Administrator tilføje dig som 'Collaborator'
  2. Installer git værktøjer fra git-scm.com/download. Click på link eller åben din browser og gå til . Her kan du downloade programmet og dernæst åbne det for at installere.
  3. Alternativt download GitHub Desktop git desktop, hvis du hellere vil anvende GUI fremfor kommando-linje... Click på link eller åben din browser og gå til. Her kan du downloade programmet og dernæst åbne det for at installere.
  4. Du skal nu lave en arbejdskopi af hjemmesiderne på din PC, hvor du kan lave ændringer, som du afsluttende sender tilbage / 'commiter', efter at du har lavet dine ændringer. Du kan 'clone' hjemmesiderne til din PC, hvis du følger starten i Desktop programmet. Hver gang du skal lave ændringer, kan du først clicke på "Fetch Origin" og dernæst "Pull origin", om andre har lavet ændringer i mellemtiden, så du har seneste filer. Hvis du anvender kommando-linje, så kør denne kommando fra den sti, hvor du vil have den lokale kopi:

      5. F.eks. : "git clone https://github.com/oresundstrandpark-etape2/oresundstrandpark-etape2.github.io.git"

  5. Efter hver ændring skal du så lave 'commit til master' og dernæst push - som gemmer dine ændringer til github. Du kan dernæst checke hjemmesiderne kort tid efter og se, om dine ændringer ser ud som forventet.

Dit brugerid på github skal sættes op til at være 'collaborator' før du kan tilføje dokumenter. Kontakt web administrator for at blive tilføjet.


Følgende trin for at uploade et nyt referat:

For pdf upload

    • placer pdf-filen i dokumenter-folderen som eksempelvis referat_YYYYMMDD.pdf

For word til html

  • Brug https://word2cleanhtml.com/ til at konvertere Word til HTML
  • Åben web-sitet, og dernæst Word-dokumentet. Markere alt i Word og copy-paste til web-sitet
  • Tryk 'Convert to clean html', og kopier indholdet tilbage til en ny html-fil, eksempelvis kaldet dokumenter\referat_YYYYMMDD.html
  • Kopier html-header fra referat_20161101.html til nyt dokument
  • Sikre, at Encoding er UTF8, så æøå vises korrekt (brug eksempelvis NotePad++ værktøjet)

Opdater index.html

    • Tilføj en sektion til index.html med link til den nye referat_YYYYMMDD.html fil

Opdater referat.html

    • Tilføj en linje til referat.html med link til den nye referat_YYYYMMDD.html fil

Upload to github

    • Kør a.bat {kommentar} eller .\a.sh (mac/linux), eksempelvis "a.bat referat1nov"; eller brug git kommandoer (git add, git commit, git push som omtalt i andre kilder)

Post-check

    • Check hvordan det ser ud på f.eks. : https://oresundstrandpark-etape2.github.io - det tager et par minuter efter upload før siderne er opdaterede.

Ny hurtigere metode

Kræver installation af Python (https://www.python.org/downloads/) og Pandoc (https://pandoc.org/installing)

1. Placer Word og PDF dokumenter i Publish folder

2. Kør kommando : python publish.py EF-2-moede-07012019 20190107

3. Kør a.bat {kommentar} eller .\a.sh (mac/linux), eksempelvis "a.bat referat1nov"; eller brug git kommandoer (git add, git commit, git push som omtalt i andre kilder)

4. Check hvordan det ser ud på f.eks. : https://oresundstrandpark-etape2.github.io


Visual Editor

        Som gratis visual HTML editor kan anbefales : BlueGriffon - hent den fra http://bluegriffon.org.

        Check altid filer for UTF-8 format/encoding, ved brug af Notepad++ - samt for danske tegn.

        Check eksempelvis sider, når de er publiceret.
